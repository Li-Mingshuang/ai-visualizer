# 排序算法总结

## 交换类

每轮通过交换元素将目标值移到正确边界。

**冒泡**：每轮在无序区边扫边交换，最大值冒到右边界。可提前终止（一轮无交换即有序）。

```python
def bubble_sort(arr):
    n = len(arr)
    for i in range(n):
        swapped = False
        for j in range(n - i - 1):
            if arr[j] > arr[j + 1]:
                arr[j], arr[j + 1] = arr[j + 1], arr[j]
                swapped = True
        if not swapped:
            break
    return arr
```

**鸡尾酒排序**：冒泡的改进，双向扫描——先从左到右把最大值冒到右边，再从右到左把最小值沉到左边，对部分有序数据收敛更快。

```python
def cocktail_sort(arr):
    lo, hi = 0, len(arr) - 1
    while lo < hi:
        swapped = False
        for i in range(lo, hi):
            if arr[i] > arr[i + 1]:
                arr[i], arr[i + 1] = arr[i + 1], arr[i]
                swapped = True
        hi -= 1
        for i in range(hi, lo, -1):
            if arr[i] < arr[i - 1]:
                arr[i], arr[i - 1] = arr[i - 1], arr[i]
                swapped = True
        lo += 1
        if not swapped:
            break
    return arr
```

**快排**：分治，选 pivot 将数组分为两侧原地排序。平均 O(n log n)，最坏（pivot 每次最大/最小）退化 O(n²)。递归栈空间 O(log n)。

```python
def quick_sort(arr, lo=0, hi=None):
    if hi is None:
        hi = len(arr) - 1
    if lo < hi:
        p = partition(arr, lo, hi)
        quick_sort(arr, lo, p - 1)
        quick_sort(arr, p + 1, hi)
    return arr

def partition(arr, lo, hi):
    pivot = arr[hi]
    i = lo - 1
    for j in range(lo, hi):
        if arr[j] <= pivot:
            i += 1
            arr[i], arr[j] = arr[j], arr[i]
    arr[i + 1], arr[hi] = arr[hi], arr[i + 1]
    return i + 1
```

## 插入类

维护有序区，每次将新元素插入有序区的正确位置。

**插入**：维护左侧有序区，每次取右侧一个元素插入有序区正确位置。近似有序时接近 O(n)。

```python
def insertion_sort(arr):
    for i in range(1, len(arr)):
        key = arr[i]
        j = i - 1
        while j >= 0 and arr[j] > key:
            arr[j + 1] = arr[j]
            j -= 1
        arr[j + 1] = key
    return arr
```

**折半插入**：插入排序的改进，用二分查找找插入位置，比较次数降到 O(log n)，但移动次数不变仍是 O(n²)。

```python
def binary_insertion_sort(arr):
    for i in range(1, len(arr)):
        key = arr[i]
        lo, hi = 0, i
        while lo < hi:
            mid = (lo + hi) // 2
            if arr[mid] <= key:
                lo = mid + 1
            else:
                hi = mid
        for j in range(i, lo, -1):
            arr[j] = arr[j - 1]
        arr[lo] = key
    return arr
```

**希尔**：插入排序的改进，先按大步长分组做插入排序，逐步缩小步长，最后步长为1。大步长时元素大跨度移动，整体接近有序后再精排，避免插入排序小步长挪动的低效。

```python
def shell_sort(arr):
    n = len(arr)
    gap = n // 2
    while gap > 0:
        for i in range(gap, n):
            key = arr[i]
            j = i
            while j >= gap and arr[j - gap] > key:
                arr[j] = arr[j - gap]
                j -= gap
            arr[j] = key
        gap //= 2
    return arr
```

## 选择类

每轮从无序区选出最值，放到边界位置。

**选择**：每轮扫无序区找最小值位置，最后一次交换到边界。每轮最多1次交换，但无法提前终止。

```python
def selection_sort(arr):
    n = len(arr)
    for i in range(n):
        min_idx = i
        for j in range(i + 1, n):
            if arr[j] < arr[min_idx]:
                min_idx = j
        arr[i], arr[min_idx] = arr[min_idx], arr[i]
    return arr
```

**堆排**：建堆 O(n)（从下往上 sift-down，底层节点多但代价小，级数收敛），然后每次取堆顶 O(log n)。原地排序，空间 O(1)。

```python
def heap_sort(arr):
    n = len(arr)
    for i in range(n // 2 - 1, -1, -1):  # 建堆
        sift_down(arr, i, n)
    for i in range(n - 1, 0, -1):         # 取堆顶
        arr[0], arr[i] = arr[i], arr[0]
        sift_down(arr, 0, i)
    return arr

def sift_down(arr, i, n):
    largest = i
    l, r = 2 * i + 1, 2 * i + 2
    if l < n and arr[l] > arr[largest]:
        largest = l
    if r < n and arr[r] > arr[largest]:
        largest = r
    if largest != i:
        arr[i], arr[largest] = arr[largest], arr[i]
        sift_down(arr, largest, n)
```

## 分治类

递归拆分后合并，复杂度恒定。

**归并**：分治，递归拆成单元素后合并。合并需要额外 O(n) 空间，稳定且复杂度恒定。

```python
def merge_sort(arr):
    if len(arr) <= 1:
        return arr
    mid = len(arr) // 2
    left = merge_sort(arr[:mid])
    right = merge_sort(arr[mid:])
    return merge(left, right)

def merge(left, right):
    result = []
    i = j = 0
    while i < len(left) and j < len(right):
        if left[i] <= right[j]:
            result.append(left[i]); i += 1
        else:
            result.append(right[j]); j += 1
    return result + left[i:] + right[j:]
```

## 非比较类

不基于元素间比较，利用数值本身的信息，突破 O(n log n) 下界，但限于整数且值域有限。

**计数**：统计每个值出现次数，适合值域范围 k 不大的整数。

```python
def counting_sort(arr):
    if not arr:
        return arr
    offset = min(arr)
    k = max(arr) - offset + 1
    count = [0] * k
    for x in arr:
        count[x - offset] += 1
    result = []
    for i, c in enumerate(count):
        result.extend([i + offset] * c)
    return result
```

**基数**：按位数从低到高多轮计数排序，适合多位整数。

```python
def radix_sort(arr):
    max_val = max(arr)
    exp = 1
    while max_val // exp > 0:
        arr = counting_sort_by_digit(arr, exp)
        exp *= 10
    return arr

def counting_sort_by_digit(arr, exp):
    count = [0] * 10
    for x in arr:
        count[(x // exp) % 10] += 1
    for i in range(1, 10):
        count[i] += count[i - 1]
    result = [0] * len(arr)
    for x in reversed(arr):
        idx = (x // exp) % 10
        count[idx] -= 1
        result[count[idx]] = x
    return result
```

## 关键对比

**冒泡 vs 选择**：都是每轮确定一个边界元素，但冒泡边扫边交换（O(n) 次交换/轮），选择只在最后交换一次。冒泡用更多交换换来提前终止的可能，是"用平均换最优"的 trade-off。

**冒泡 vs 插入**：冒泡和无序区比较，插入和有序区比较。冒泡靠交换，插入靠移动。插入在近似有序数据上更快。

**鸡尾酒 vs 冒泡**：双向扫描使每轮同时确定最大值和最小值的位置，对"大部分有序但两端有少量乱序"的数据收敛轮数减半。

**希尔 vs 插入**：希尔是插入的泛化，步长为1时退化为普通插入。希尔用大步长解决了插入排序"小步慢挪"的问题，但破坏了稳定性。

**折半插入 vs 插入**：只优化了比较次数，移动次数不变，时间复杂度仍是 O(n²)。适合比较代价远大于移动代价的场景。

**选择 vs 堆排**：都是选择类，每轮取最值。堆排用堆维护无序区，将每轮找最值的代价从 O(n) 降到 O(log n)。

**快排 vs 归并**：快排原地 O(log n) 空间，但最坏 O(n²)；归并稳定且复杂度恒定，但需要 O(n) 额外空间。

**快排 vs 堆排**：快排平均更快（cache 友好），堆排最坏也是 O(n log n) 且空间 O(1)。

**插入 vs 归并**：两者都适合近似有序数据，但插入最坏 O(n²)，归并复杂度恒定 O(n log n)。数据量小时插入常数小更快，数据量大时归并更稳。

**堆排 vs 归并**：堆排空间 O(1) 但不稳定，归并稳定但需要 O(n) 额外空间。需要稳定性时选归并，内存敏感时选堆排。

**计数/基数 vs 比较排序**：比较排序的下界是 O(n log n)（决策树高度），无法突破。计数和基数不基于比较，利用数值本身的信息，能做到 O(n)——但代价是限制了适用范围（整数、值域有限）。
